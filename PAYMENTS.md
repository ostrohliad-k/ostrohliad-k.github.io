# Підключення оплати бронювання (Monobank Acquiring)

Сторінка `booking.html` дозволяє обрати дату й час і сплатити передоплату
(зараз **1000 грн**, налаштовується через `BOOKING_DEPOSIT` у `generate.py`).

Кнопка «Сплатити та забронювати» працює у трьох режимах — автоматично
обирається залежно від налаштувань:

1. **Monobank Acquiring (обрано)** — якщо заповнено `BOOKING_API_URL`.
   Кнопка створює реальний інвойс і веде клієнта на сторінку оплати Monobank.
2. **Статичне посилання** — якщо заповнено `PAYMENT_LINK` (резервний варіант).
3. **Без оплати** — відкриває Instagram Direct для підтвердження вручну.

---

## Кроки для Monobank Acquiring

### 1. Отримати токен мерчанта
- Підключіть інтернет-еквайринг Monobank: <https://web.monobank.ua/> →
  розділ **Acquiring / Інтернет-еквайринг**.
- Скопіюйте **токен мерчанта** (X-Token).

### 2. Задеплоїти бекенд
У репозиторії вже є функція: `api/create-invoice.js` (формат Vercel).

Найпростіший шлях — **Vercel** (безкоштовно, можна хостити і сам сайт):
1. Створіть акаунт на <https://vercel.com> і залийте цю папку (через GitHub або `vercel` CLI).
2. У налаштуваннях проєкту → **Environment Variables** додайте:
   - `MONOBANK_TOKEN` = ваш токен мерчанта
   - `SITE_URL` = адреса сайту, напр. `https://your-site.vercel.app` (для повернення після оплати)
   - `WEBHOOK_URL` = (необов'язково) адреса для статусів оплати
3. Після деплою endpoint буде доступний за адресою:
   `https://your-site.vercel.app/api/create-invoice`

> Інші платформи: код стандартний (Web Fetch). Для Netlify Functions /
> Cloudflare Workers потрібна мінімальна адаптація обгортки `handler`.

### 3. Вписати URL у генератор
У `generate.py` вгорі:
```python
BOOKING_API_URL = "https://your-site.vercel.app/api/create-invoice"
```
Потім перегенеруйте сайт:
```bash
python generate.py
```

### 4. Перевірити
Відкрийте `booking.html`, оберіть дату й час, натисніть «Сплатити та забронювати» —
має відкритися сторінка оплати Monobank. Після оплати клієнта повертає на
`booking.html?paid=1` із повідомленням-подякою.

---

## Безпека
- Токен `MONOBANK_TOKEN` зберігається **лише** на сервері (env var) і ніколи не
  потрапляє у HTML/JS. Не комітьте його в репозиторій.
- Сума передоплати фіксується на бекенді (мінімум 1000 грн), тож її не можна
  підмінити з боку клієнта.

## Налаштування суми / годин
У `generate.py`:
- `BOOKING_DEPOSIT` — сума передоплати, грн.
- `BOOKING_HOURS` — робочі години, напр. `(10, 19)`.
- `BOOKING_STEP_MIN` — крок слотів, хв.
